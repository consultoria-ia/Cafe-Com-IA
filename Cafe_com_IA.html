Fundação Lemann
Skill
Quando usar

A descrição é o gatilho.

Ela diz ao Codex em quais situações a skill deve ser carregada.

Regra

Descrição boa é específica o bastante para evitar uso errado.

Como agir

O fluxo reduz ambiguidade.

  • Identificar público, duração e objetivo.
  • Criar agenda com teoria, demonstração e prática.
  • Gerar checklist e próximos passos.
O que entregar

O formato vira padrão.

Para iniciantes, a skill precisa dizer o que evitar: jargão, excesso de abstração e respostas sem exemplo.

Fundação Lemann
Plugin
Parte 3

Plugin é o pacote que organiza capacidades.

Para iniciantes, comece simples: um plugin com uma skill clara. Depois adicione assets, scripts e integrações quando houver necessidade real.

Skill→ Plugin→ Instalar→ Reutilizar
Dentro do pacote

O plugin transforma conhecimento em produto interno.

  • Skills para procedimentos reutilizáveis.
  • Assets para templates, exemplos e materiais.
  • Scripts para tarefas repetitivas ou frágeis.
  • Integrações quando o fluxo precisar conversar com outras ferramentas.

Antes e depois
Prompt solto
Funciona uma vez
  • ×Depende de repetir contexto toda vez.
  • ×O padrão muda de acordo com o prompt do dia.
  • ×Difícil de compartilhar com outra pessoa.
Risco

O conhecimento fica preso na conversa, não no sistema.

Skill + plugin + automação
Vira capacidade recorrente
  • ✓O comportamento fica documentado e reutilizável.
  • ✓O plugin organiza a capacidade como pacote.
  • ✓A automação executa a rotina sem novo pedido manual.
Ganho

O time passa a operar com padrões, não só com prompts.

Mensagem central: prompt resolve uma tarefa; skill cria padrão; plugin empacota capacidades; automação cria rotina.

Fundação Lemann
Automações
Parte 4

Automação é a IA como rotina combinada.

A automação boa explicita frequência, tarefa, contexto e formato esperado.

Modelo

Todo [frequência], use a skill [nome] para [tarefa] e entregue [formato esperado].

  • Todo dia às 9h: revisar pendências do evento.
  • Na véspera: gerar roteiro, materiais e mensagem aos participantes.
  • Toda sexta: resumir aprendizados e sugerir melhorias.
Fundação Lemann
Prática final
Exercício

Cada pessoa sai com um miniassistente pessoal.

A entrega final é pequena, concreta e apresentável em dois minutos.

1. ProblemaEscolha uma tarefa recorrente: reunião, aula, evento, comunicação ou relatório.
2. SkillEscreva quando usar, fluxo de trabalho, estilo e formato de saída.
3. PluginEmpacote a skill com nome, descrição e estrutura simples.
4. AutomaçãoDefina a rotina: frequência, contexto e entrega esperada.
Fechamento

A melhor skill não é a mais longa. É a que reduz ambiguidade.

Projeto organiza o trabalho. Skill ensina o comportamento. Plugin empacota capacidades. Automação cria rotina.

# Preparador De Eventos

Ajude a transformar uma ideia de evento em materiais práticos.

## Fluxo

1. Identifique público, duração e objetivo.
2. Crie uma agenda clara.
3. Separe teoria, demonstração e prática.
4. Gere checklist de preparação.
5. Sugira materiais de apoio.
6. Termine com próximos passos objetivos.

## Estilo

- Use linguagem simples.
- Priorize exemplos práticos.
- Evite jargão técnico sem explicação.
- Para iniciantes, explique um conceito por vez.
